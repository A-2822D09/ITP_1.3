ITP 1.3 のコマンドの一覧

ITP 1.3 のコマンドの一覧です。構文の記号の意味は下記の通りです。

() | 必須では無いものです。

<> | 複数の選択肢がある、または変数です。

* | 特別な例外や注記があります。

time (/<arguments>)

時刻を表示します。

引数を指定しなかった場合、出力は YYYY-MM-DDTHH:MM:SS.mmmZ 形式で行われます。例: 2023-12-09T17:49:00.000Z

arguments

言語コードを指定してください。例: /ja, /en-us

ITP 1.0 ~

time

time /ja

time /en-us

eval `<text>` /<arguments>

入力された文字列をJavaScriptのコードとして解釈し、実行します。 このコマンドの使用は推奨しません。

text

JavaScriptのコードとして解釈する文字列です。

arguments

実行をフロント側かサーバー側で行うかを指定します。

値は front または server のみ使用できます。

ITP 1.0 ~

eval `alert("hello world!")` /front

eval `new Date().toLocaleString("ar")` /server

echo `<text>`

入力された文字列を表示します。

バッククォートで囲われた文字列以外は無視されます。

text

表示したい文字列です。文字参照やASCII文字列( \n など)は使用できません。

ITP 1.0 ~

echo `Hello world!`

このような記述もできますが、混乱を招く可能性があるためおすすめしません。

ITP 1.0 ~

echo `Hello world!\n` // => Hello world!\n

reload

ページを再更新し、リセットします。

bgcolor <HEXcode>

背景色を変更します。

HEXcode

3桁の HEX カラーコードです。例: #000

ITP 1.0 ~

bgcolor #000

ITP 1.0 ~

bgcolor #000000

bgcolor rgb(0, 0, 0)

color <HEXcode>

文字色を変更します。

HEXcode

3桁の HEX カラーコードです。例: #fff

ITP 1.0 ~

color #000

ITP 1.0 ~

color #000000

color rgb(0, 0, 0)

help

ヘルプを表示します。

encode `<text>` /<arguments>

入力された文字列をエンコードします。エンコードの形式は引数で指定します。

text

エンコードする文字列です。

arguments

エンコードの形式です。現在使用できるエンコード形式は下記の通りです。

asc2 (ASCII エンコード) , uri (URI エンコード)

ITP 1.1 ~

encode `Hello world!` /asc2// => SGVsbG8gd29ybGQh

encode `あいうえお` /uri// => %E3%81%82%E3%81%84%E3%81%86%...

decode `<text>` /<arguments>

入力された文字列をデコードします。デコードの形式は引数で指定します。

text

デコードする文字列です。

arguments

デコードの形式です。現在使用できるデコードの形式は下記の通りです。

asc2 (ASCII デコード) , uri (URI デコード)

ITP 1.1 ~

decode `SGVsbG8gd29ybGQh` /asc2// => Hello world!

decode `%E3%81%82%E3%81%84%E3%81%86%...` /uri// => あいうえお

commandindex

このコマンドの一覧を開きます。

open `<URL>`

入力されたURLを開きます。

※注意: JavaScriptの仕様上、open `www.example.com` のように入力すると、(URL)/www.example.com/ に遷移されてしまいます。
これを回避するためにURLの先頭にはhttp:// または https:// を付けてください。

ITP 1.2 ~

open `https://www.example.com`

ITP 1.2 ~

open `www.example.com`// => (URL)/www.example.com

useragent /<arguments>

ユーザーエージェントを表示します。

arguments

アクセス時のユーザーエージェントを表示する際は log を、現在のユーザーエージェントを表示する際は now を指定してください。

ITP 1.2 ~

useragent /log

useragent /now

commandhistory

コマンドの履歴を表示します。履歴の保持は本ページを閉じるまで有効です。

savehistory

コマンドの履歴をcookieに保存します。有効期間は3日です。
